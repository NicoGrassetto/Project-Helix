# Contributing to Project Helix

Thanks for your interest in contributing.

## Ways to contribute

- Report bugs
- Propose features or documentation improvements
- Submit fixes and enhancements via pull requests

## Before you start

1. Open an issue to discuss substantial changes before implementing them.
2. Check existing issues and pull requests to avoid duplicate work.
3. Keep changes focused and small when possible.

## Development setup

1. Fork and clone the repository.
2. Install dependencies from `dependencies/requirements.txt`.
3. Validate your environment with `notebooks/getting-started.ipynb`.
4. Run or test changes locally (for example with `src/train.py` or job scripts under `jobs/`).

## Pull request guidelines

1. Create a branch for your change.
2. Add a clear description of the problem and your solution.
3. Link related issues in the PR description.
4. Include updates to docs when behavior or usage changes.
5. Keep PRs reviewable; split large work into multiple PRs if needed.

## Coding and documentation expectations

- Follow existing project structure and naming conventions.
- Prefer clear, maintainable code over clever shortcuts.
- Update README/docs for user-facing changes.

## Security

Do not open public issues for security vulnerabilities.

Report vulnerabilities privately via GitHub Security Advisories:
https://github.com/nicograssetto/Project-Helix/security/advisories
